Diagnosis

Error type: TypeScript error TS1149 (file name differs only in casing / duplicate inclusion).
Cause: A file is being referenced with different casings/paths so TypeScript ends up including the same physical file under two different names. The error originates in artifacts/portfolio/src/components/ui/alert-dialog.tsx (see line 88). The file imports "@/components/ui/button" (see line 91), but the actual file path/casing on disk does not match that import exactly. This passes on case-insensitive dev machines (macOS/Windows) but fails on Netlify's Linux CI (case-sensitive).
